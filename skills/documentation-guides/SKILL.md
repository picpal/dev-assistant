# Documentation Guides Skill

Standards and templates for code documentation across Java, Python, and TypeScript.

## Purpose

Ensure consistent, helpful documentation that improves code maintainability and developer onboarding.

## Coverage

- **JavaDoc**: Java class and method documentation
- **Sphinx/Docstrings**: Python documentation standards
- **JSDoc/TSDoc**: JavaScript/TypeScript documentation

## Reference Files

- `javadoc-standards.md`: JavaDoc conventions and examples
- `sphinx-standards.md`: Python docstring formats (Google, NumPy, Sphinx styles)
- `jsdoc-standards.md`: JSDoc/TSDoc patterns for TypeScript

## Key Principles

1. Document why, not what (code shows what)
2. Include examples for complex APIs
3. Document parameters, returns, and exceptions
4. Keep documentation up-to-date with code
5. Use consistent formatting

## Integration with Doc-Reference Agent

Provides standards that the `doc-reference` agent uses when searching and presenting documentation.
